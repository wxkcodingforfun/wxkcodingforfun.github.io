第1章 计算机概要与技术

计算机思想

  1. 通过冗余提高可靠性
  2. 通过并行提高性能
  3. 通过流水线提高性能
  4. 通过预测提高性能
  5. 加速大概率事件

虽然有八个思想，但是我觉得上面的这五个是编程的时候用得到的，别是流水线。

一个程序的性能取决于四个因素：

算法，编程语言·编译器·体系结构，处理器和存储系统，I/O系统（硬件和操作系统）

编译程序的流程

​ 从高级语言到硬件语言：

​ 计算机硬件语言就是一些二进制数。计算机会识别特殊的二进制数来当做指令。例如，位串 1000110010100000 告诉计算机将两个数相加。

​ 后面，出现了汇编程序，因为这些二进制数不好记。所以最开始的开发人员，开发了汇编，用来将一些符号翻译成对应的二进制。例：add A, B -> 1000110010100000。

​ 但是汇编语言由于太接近计算机硬件语言，使用汇编语言来实现一些复杂操作实在太难。所以有了高级编程语言通过编译程序转换到汇编的过程。

​ 所以一个程序由int main()到执行的过程，是先由编译器将字符翻译成汇编语言程序，然后再由汇编变成二进制机器语言。这个过程中，由于执行程序的硬件不同，这些硬件识别的指令集也不同，所以我们需要根据不同的平台来选特定的编译器。

比如：使用CISC(Complex Instruction Set Computers)复杂指令集的AMD的x86和Intel芯片，还有使用RISC(Reduced Instruction Set Computers)精简指令集的ARM，IBM Power芯片。

CISC与RISC对比

CSIC RISC
复杂指令的执行需要更多的时钟周期 简单指令只需1个时钟周期
所有指令都可访问内存 只有loads/stores指令可访问内存
寄存器少 寄存器多
微代码翻译模板复杂 软件编译器复杂
指令多，模式多 指令少，模式少
指令格式可变 指令格式固定
指令由微代码翻译执行 指令直接由硬件执行
无流水线或流水线程度较低 流水线结构

​ 相同的程序代码，想要在不同的平台上跑起来。首先需要选用对应平台的编译器，因为不同平台使用的汇编语言指令集不一样，这样就导致翻译的二进制机器语言也不一样。比如一个test.c需要通过gcc编译器将程序编译成CISC的指令集汇编语言，才能拿在CISC的CPU上运行。

2.12 翻译并执行程序

单独编译和汇编每个过程，使得某一行代码的改变只需要编译和汇编一个过程。

链接器(Linker)，把所有独立汇编的机器语言程序”拼接”在一起。

链接器的三步工作：

  1. 将代码和数据模块象征性的放入内存
  2. 决定数据和指令标签的地址。
  3. 修补内部和外部的引用。

采用链接器的原因是 修补代码比重新编译和汇编要快很多。

加载器

在UNIX系统中，加载器工作流程：

  1. 读取可执行文件头来确定代码段和数据段的大小

  2. 为正文和数据创建一个足够大的地址空间。

  3. 将可执行文件中的指令和数据复制到内存中。

  4. 把主程序的参数（如果存在）复制到栈顶

  5. 初始化机器寄存器，将栈指针指向第一个空位置。

  6. 跳转到启动例程，它将参数复制到参数寄存器并且调用程序的main函数。当main函数返回时，启动例程通过系统调用exit中止程序。
